# Star-rectangle first appearance

## Status

State-independent first-appearance theorem for the repeated-state collision
cells produced by the heavy completion-step reference reserve.

The theorem shows that once one deterministic base reference is retained, each
collision cell has a unique ordered four-point rectangle configuration. Thus
state multiplicity is converted without loss into a configuration union. The
remaining problem is projection of those configurations onto reusable physical
pair resources.

---

## 1. One repeated-state cell

Let a step state `S` repeat at references

```math
c_0<c
```

with one fixed orientation

```math
\sigma\in\{-1,+1\}.
```

For one step `d in S`, the parent contains

```math
c_0+\sigma d,
\qquad
c_0+2\sigma d,
\qquad
c+\sigma d,
\qquad
c+2\sigma d.
```

Define the ordered star-rectangle token

```math
\mathfrak r(c_0,c,\sigma,d)
=
\bigl(
 c_0+\sigma d,
 c_0+2\sigma d,
 c+\sigma d,
 c+2\sigma d
\bigr).
```

Its weight is

```math
w(\mathfrak r)=\frac1d.
```

This is exactly the weight of the repeated step occurrence represented by the
cell.

---

## 2. Injectivity

Write

```math
\mathfrak r=(a_1,a_2,b_1,b_2).
```

Then

```math
c_0=2a_1-a_2,
\qquad
c=2b_1-b_2,
```

and

```math
\sigma d=a_2-a_1=b_2-b_1.
```

Since `d>0`, the sign determines `sigma` and the absolute value determines
`d`. Therefore

```math
\boxed{
\mathfrak r(c_0,c,\sigma,d)
=
\mathfrak r(c_0',c',\sigma',d')
\Longrightarrow
(c_0,c,\sigma,d)
=
(c_0',c',\sigma',d').
}
```

Every collision cell has one unique configuration token.

---

## 3. Exact collision identity

For one repeated oriented state `S` with reference set `C`, choose

```math
c_0=\min C.
```

The repeated-state collision load is

```math
(|C|-1)H(S)
=
\sum_{c\in C\setminus\{c_0\}}
\sum_{d\in S}
\frac1d.
```

By injectivity,

```math
\boxed{
(|C|-1)H(S)
=
\sum_{c\in C\setminus\{c_0\}}
\sum_{d\in S}
w(\mathfrak r(c_0,c,\sigma,d)),
}
```

with no repeated configuration token inside the sum.

Summing over all repeated-state families remains collision-free at the full
ordered-configuration level, provided the state/reference family identifier is
chosen by physical first appearance.

---

## 4. Rectangle coordinates

Each star rectangle has side differences

```math
\delta=c-c_0,
\qquad
d.
```

The four points may be written

```math
x,
\quad
x+\sigma d,
\quad
x+\delta,
\quad
x+\delta+\sigma d,
```

in the first copy, together with the doubled-step relation already encoded by
the ordered roles.

The abstract aspect token is

```math
(\delta,d,k,\sigma),
\qquad
2^kd\le\delta<2^{k+1}d.
```

Several distinct absolute rectangles may project to the same abstract aspect
token. That reuse is a separate, explicitly measurable collision layer.

---

## 5. Pair projections

A rectangle token exposes several physical parent-root pairs, including:

```text
two vertical target pairs of gap d;
two reference pairs of gap delta;
first-copy and second-copy horizontal pairs generated by internal step pairs;
diagonal pairs with gaps |delta-d| and delta+d when present.
```

The configuration is unique, but individual projected pairs can be shared by
other configurations. Therefore configuration injectivity does not authorize
independent summation of all projected pair energies.

The next allocation map must be

```math
\mathfrak r
\longmapsto
\text{one or more projected physical pair tokens}
```

with source-weighted first appearance and an explicit collision remainder.

---

## 6. Relation to the four-ratio exclusion

For two steps with difference `r`, the double-translate four-ratio theorem
excludes

```math
\delta/r\in\{1/2,1,2,4\}.
```

Thus the configuration ledger has exact structural zeroes in four aspect
classes. The star-rectangle token retains enough information to apply those
zeroes before projecting to a scalar reference difference or pair gap.

---

## 7. Consequence

The repeated numerical-state count is not the correct collision statistic.
The correct hierarchy is

```text
abstract step-state occurrence;
ordered affine double lift;
star-rectangle configuration first appearance;
abstract aspect-token projection;
physical pair projection.
```

The first three layers are now exact and injective. The unresolved reuse begins
only at the last two projections.
